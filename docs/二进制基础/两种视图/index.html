两种视图

2025-03-14
1 min read

链接视图

由 .text、.data、.bss、.rodata、.symtab、.strtab 等 Section 组成。

这些 Section 在链接阶段用于解析符号、重定位代码等。

运行时并不直接使用 Section，而是使用 Segments。

执行视图

常见的 Segment：

  • LOAD：加载到内存的段，包含代码（R-X）、数据（RW-）。
  • DYNAMIC：用于动态链接的信息。
  • INTERP：指向动态链接器（如 /lib/ld-linux.so.2）。
  • NOTE：包含程序的额外信息，如 ABI 版本等。

程序编译过程

预处理

  1. 处理 #include 指令，替换头文件内容。
  2. 处理 #define 宏定义，进行文本替换。
  3. 处理 #ifdef / #ifndef 等条件编译指令。
  4. 删除 注释（// 或 /* ... */）。
  5. 生成 纯代码（无预处理指令）。
gcc -E main.c -o main.i

编译

  1. 将预处理后的 C 代码（main.i）翻译成 汇编代码（Assembly）。
  2. 语法分析（Parsing），检查代码语法是否正确。
  3. 词法分析（Lexical Analysis），将源代码转换为 Token。
  4. 语义分析（Semantic Analysis），分析变量作用域、类型匹配等。
  5. 生成 优化后的汇编代码。
gcc -S main.i -o main.s

汇编

  1. 将汇编代码（.s 文件）转换为 机器指令（Object Code，目标文件）。
  2. 生成 二进制的 .o（目标文件），但此时的代码 仍无法独立运行。
gcc -c main.s -o main.o

链接

  1. 合并多个目标文件（.o 文件）。
  2. 解析 外部符号（比如 printf 来自 libc）。
  3. 链接 静态库（\*.a）或动态库（\*.so）。
  4. 生成最终的 可执行文件。
gcc main.o -o main

静态链接：将库的代码直接嵌入可执行文件，文件较大，但运行时不依赖外部库。

动态链接：运行时加载 .so 文件（如 libc.so），文件较小，但依赖外部库。
